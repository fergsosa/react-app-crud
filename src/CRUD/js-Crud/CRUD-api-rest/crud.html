¿Qué es una API REST?

🔻

Una API REST, o API RESTful, es una forma de diseñar interfaces de programación de aplicaciones (API) que siguen los principios de la arquitectura REST (Representational State Transfer).
En términos más simples, es un conjunto de reglas que permiten que diferentes aplicaciones se comuniquen entre sí de manera eficiente y estándar.

¿Para qué sirve una API REST?

🔻

Imagina que quieres construir una aplicación móvil que muestre el clima.
En lugar de recopilar toda la información meteorológica tú mismo, puedes utilizar una API REST proporcionada por un servicio meteorológico.
Esta API te permitirá solicitar datos sobre el clima en cualquier lugar del mundo y tu aplicación mostrará esa información a tus usuarios.

Características principales de una API REST:

🔻
  • Basada en HTTP: Utiliza los métodos HTTP (GET, POST, PUT, DELETE, etc.) para realizar operaciones sobre los recursos.
  • Stateless (Sin estado): Cada solicitud contiene toda la información necesaria para ser procesada, no se almacena estado en el servidor entre solicitudes.
  • Recursos: Organiza la información en recursos identificables mediante una URL única .
  • Formato de Datos: generalmente utiliza JSON o XML para el intercambio de datos, siendo JSON el formato más común.
  • Interfaces uniformes: Utiliza un conjunto limitado de operaciones (GET, POST, PUT, DELETE) para manipular los recursos.
  • Códigos de estado HTTP:La API REST responde con códigos de estado HTTP que indican el resultado de la solicitud
    1. 200 OK: La solicitud fue exitosa.
    2. 201 Created: Se creó un nuevo recurso.
    3. 404 Not Found: El recurso no se encontró.
    4. 500 Internal Server Error: Ocurrió un error en el servidor.
Ventajas de las API REST:

🔻
  • Simplicidad: Son fáciles de entender y usar, lo que facilita su desarrollo e integración.
  • Flexibilidad: Permiten crear aplicaciones distribuidas y escalables.
  • Independencia de la plataforma: Pueden ser consumidas desde cualquier plataforma o lenguaje de programación.
  • Ampliamente utilizadas: Existe una gran comunidad y muchos recursos disponibles.
Ejemplo: API REST de una lista de tareas

🔻
  • GET /tareas: Obtener todas las tareas.
  • POST /tareas: Crear una nueva tarea.
  • GET /tareas/{id}: Obtener una tarea específica por su ID.
  • PUT /tareas/{id}: Actualizar una tarea.
  • DELETE /tareas/{id}: Eliminar una tarea.
